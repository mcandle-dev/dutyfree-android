## 1. 시스템 범위 개요 (Scope Summary)

> 기술 상세(BLE 인터페이스, 패킷 포맷, 기능 요구사항)는 **PRD_DutyFree_BLE_v1.4.md** 참조

### 1.1 면세 판매 표준 절차 (고객 경험 관점)

면세 판매는 **반드시 신원/자격 확인이 상품 스캔보다 먼저** 이루어져야 합니다.
본 BLE 시스템은 이 절차를 **자동화**하여 고객과 직원의 경험을 개선합니다.

```
Step 1.  신원 및 자격 확인  →  BLE 앱이 자동 처리  ✅
Step 2.  상품 바코드 스캔   →  자격 확인 완료 후 진행
Step 3.  결제              →  별도 카드 단말기 처리 (앱 범위 외)
```

### 1.2 BLE 앱이 개선하는 경험 구간

```
                  [BLE 앱 담당 구간]
                  ╔══════════════════════════════════╗
고객 접근 → BLE 연결 → 신원 자동 수신 → 자격 조회 → 상품 스캔 → 금액 확정
                  ╚══════════════════════════════════╝
                                                           ↓
                                                  [별도 카드 단말기 모듈]
                                                  오프라인 카드 결제
```

## 2. 페르소나

### 2.1 매장 직원: 박수현 (30세, 면세점 판매사원)

**배경**
- 인천공항 T1 롤렉스 면세점 근무 4년차
- 하루 평균 50~60명 고객 응대
- 면세 판매 표준 절차상 여권·탑승권 확인이 항상 첫 단계

**현재 경험 (As-Is)**
```
😰 여권 요청:   "여권과 탑승권 보여주세요" 매번 반복
😅 수기 입력:   알파벳 하나하나 POS에 직접 입력
😓 피크타임:    여권+탑승권 확인+입력에 2~3분 소요
😔 오타 위험:   잘못 입력 시 면세 처리 오류
```

**기대 경험 (To-Be)**
```
😊 "앱 카드 탭 눌러주세요" 한 마디 → 목록에 자동 표시
🎯 선택하면 여권·출국 정보 즉시 수신 → 구두 확인 한 마디
⚡ 자격 확인 → 상품 스캔 → 금액 확정 → 카드 단말기로 넘기면 끝
```

**Pain Points → Solutions**

| Pain Point | Solution |
|-----------|----------|
| 여권 수기 입력 오타 | 앱 1회 자동 전송으로 완전 제거 |
| 탑승권 확인 번거로움 | 앱 출국정보 자동 전송 (미등록 시만 수동) |
| 고객 응대 시간 과다 | Advertise → 즉시 연결 → 단번에 모든 정보 수신 |

---

### 2.2 고객: 김준호 (35세, IT 회사 재직)

**배경**
- 연 4~5회 해외 출장/여행, 면세 쇼핑 즐겨함
- 여권·탑승권 꺼내는 게 매번 번거로움
- 실물 카드 결제 익숙

**현재 경험 (As-Is)**
```
😤 계산대 오자마자 여권 요청: 짐 뒤져야 해서 불편
😕 탑승권도 꺼내야: 앱 화면 찾는 데 시간 걸려
😐 직원 입력 대기: 뒤에 줄 선 손님 눈치 보임
```

**기대 경험 (To-Be)**
```
😊 앱 카드 탭만 누르면 → 직원이 바로 찾아 선택
🎯 "맞습니다" 한 마디 → 자격 확인 완료
⚡ 상품 스캔 → 카드 결제 → 즉시 상품 수령
```

## 3. User Journey Map

### 3.1 Journey Overview

```
🛫 인천공항 T1 롤렉스 면세점
🕐 토요일 오전 11시 (출국 피크 타임)
👥 박수현 직원 · 김준호 고객
⏱️  약 2분 30초 (출국정보 등록 시) / 약 3분 (미등록 시)
```

### 3.2 Journey Phases 전체 흐름

```
[사전]  앱 여권/출국 정보 등록 (집 또는 공항 로비)
                    ↓
Phase 1:  고객 앱 Advertise → 직원 목록 선택 → BLE 연결   (25초)
                    ↓
        ┌───────────────────────────────────┐
        │        출국 정보 등록 여부         │
        └───────────────────────────────────┘
              ↓                     ↓
         [등록됨]               [미등록]
    Phase 2-A: 자동 수신      Phase 2-B: 직원 수동 입력
         (10초)               (탑승권 수령 · 30초)
              └──────────┬──────────┘
                         ↓
Phase 3:  여권/출국 정보 구두 확인 + 면세 자격 조회        (20초)
                         ↓
Phase 4:  상품 바코드 스캔  ← 자격 확인 완료 후            (15초)
                         ↓
Phase 5:  혜택 안내 + 결제 금액 확정                       (20초)
                         ↓
              ══════════════════════════════
              [BLE 앱 범위 종료]
              ══════════════════════════════
                         ↓
Phase 6:  별도 오프라인 결제 모듈 (카드 단말기)  [범위 외]
                         ↓
Phase 7:  결제 완료 → 면세 영수증 → 상품 현장 즉시 전달
```
