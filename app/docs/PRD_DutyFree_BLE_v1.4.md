## 1. 시스템 범위 및 결제 방식

### 1.1 BLE 앱 담당 범위

```
┌─────────────────────────────────────────────────────────┐
│                  본 BLE 앱 담당 범위                      │
│                                                         │
│  Phase 1: 고객 앱 Advertise → VPOS 스캔 & 목록 표시      │
│  Phase 2: 직원 고객 선택 & BLE 연결                      │
│               → 멤버십 + 여권/출국 정보 1회 수신          │
│  Phase 3: 구두 확인 + 면세 자격 검증 (자사 POS 조회)     │
│  Phase 4: 상품 바코드 스캔                               │
│  Phase 5: 혜택 안내 + 결제 금액 확정                     │
│                        ↓                                │
│               [결제 직전 완료]                            │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│           별도 오프라인 결제 모듈 (범위 외)               │
│   카드 단말기 실물 카드 결제 처리                         │
└─────────────────────────────────────────────────────────┘
```

### 1.2 결제 방식 정의

| 구분 | 지원 여부 | 비고 |
|------|----------|------|
| 오프라인 카드 결제 | ✅ 지원 | 별도 카드 단말기 모듈 처리 |
| 온라인 앱 결제 | ❌ 미지원 | 본 앱 범위 외 |
| 간편결제 (삼성/카카오페이 등) | ❌ 미지원 | 본 앱 범위 외 |

### 1.3 면세 한도 누적 구매액 확인 방식

| 시점 | 조회 가능 범위 | 시스템 |
|------|--------------|--------|
| 판매 시점 | 자사 당일 누적 구매액만 조회 가능 | 자사 POS |
| 타사 면세점 구매 | 판매 시점 조회 불가 | 없음 (고객 자진 신고 의무) |
| 입국 시점 | 모든 면세점 구매 통합 파악 | 관세청 유니패스(UNIPASS) |

> ⚠️ 고객 책임: 총 구매액 $800 초과 시 자진 신고 의무. 미신고 시 과징금 40~60%.

## 2. BLE 통신 인터페이스 정의

> BLE 패킷 크기 제약을 고려하여 모든 데이터는 Pipe(`|`) 구분 최소 포맷을 사용합니다.
> MTU 협상 결과 (EFR32BG22 기본 250 bytes / iOS 최대 185 bytes) 기준으로
> 65 bytes 단일 패킷은 1회 Write로 전송 가능합니다.

---

### 2.1 인터페이스 1 — 고객 앱 Advertise (연결 전)

고객이 앱의 카드 탭을 실행하면 BLE Advertising을 자동 시작합니다.
VPOS는 이 신호를 스캔하여 주변 고객 목록을 표시합니다.

**Advertise Data 포맷**
```
이름|전화번호끝4자리
```

**예시**
```
JOONHO KIM|5678
```

| 필드 | 설명 | 최대 길이 |
|------|------|-----------|
| 이름 | First Name + Last Name (여권 영문 이름) | 20자 |
| 전화번호 끝 4자리 | 본인 확인용 | 4자 |

**VPOS 목록 표시 예시**
```
JOONHO KIM   ****5678   신호강도 ▮▮▮▮
MINJUN PARK  ****1234   신호강도 ▮▮▯▯
```

---

### 2.2 인터페이스 2 — Connected 이후 통합 정보 1회 전송

직원이 목록에서 고객 선택 → GATT 연결 수립 직후
**멤버십 + 여권/출국 정보를 단일 패킷으로 1회 전송**합니다.

**통합 데이터 포맷**
```
회원번호|등급코드|포인트#여권번호|FirstName|LastName|만료일(YYMMDD)|항공사코드|편명|출국시간(HHMM)
```

> `#` = 멤버십 영역과 여권/출국 영역의 구분자

**예시 — 출국 정보 등록된 경우**
```
DF2023091234|V|258000#M12345678|JOONHO|KIM|300520|KE|KE123|1430
(총 65 bytes)
```

**예시 — 출국 정보 미등록인 경우 (Index P4~P6 빈값)**
```
DF2023091234|V|258000#M12345678|JOONHO|KIM|300520|||
```

**필드 상세**

| 영역 | Index | 필드 | 설명 | 예시 |
|------|-------|------|------|------|
| 멤버십 | M0 | 회원번호 | 면세점 멤버십 번호 | DF2023091234 |
| 멤버십 | M1 | 등급코드 | V / G / S / N | V |
| 멤버십 | M2 | 포인트 | 보유 포인트 (정수) | 258000 |
| 여권/출국 | P0 | 여권번호 | 영문+숫자, 최대 9자 | M12345678 |
| 여권/출국 | P1 | First Name | 여권 영문 이름 (대문자) | JOONHO |
| 여권/출국 | P2 | Last Name | 여권 영문 성 (대문자) | KIM |
| 여권/출국 | P3 | 만료일 | YYMMDD 6자리 | 300520 |
| 여권/출국 | P4 | 항공사 코드 | IATA 2자리 (빈값=미등록) | KE |
| 여권/출국 | P5 | 편명 | 항공사코드+숫자 | KE123 |
| 여권/출국 | P6 | 출국시간 | HHMM 4자리 | 1430 |

**등급 코드표**

| 코드 | 의미 |
|------|------|
| V | VIP |
| G | Gold |
| S | Silver |
| N | 일반 |

### 2.3 전체 BLE 통신 흐름 다이어그램

```
[고객 앱]                              [VPOS]
    │                                     │
    │── Advertise ────────────────────────►│  JOONHO KIM|5678
    │   (연결 전, 브로드캐스트)             │  (이름|전화번호끝4자리)
    │                                     │
    │◄── 직원이 목록에서 선택 ──────────────│
    │                                     │
    │── GATT Connect ─────────────────────►│
    │◄── Connected ────────────────────────│
    │                                     │
    │── 통합 정보 1회 Write ──────────────►│  DF2023091234|V|258000#
    │   (Write Characteristic)            │  M12345678|JOONHO|KIM|300520|KE|KE123|1430
    │                                     │  (단일 패킷 65 bytes)
    │                                     │
    │                    [이후 VPOS에서 처리]
    │                                     │  → split("#") 파싱
    │                                     │  → 구두 확인 + POS 자격 조회
    │                                     │  → 상품 바코드 스캔
    │                                     │  → 혜택 조회 + 결제 금액 확정
    │                                     │
    │                         [별도 모듈] │  → 카드 단말기 오프라인 결제
```

---

### 2.4 패킷 크기 및 MTU 근거

| 항목 | 값 | 비고 |
|------|-----|------|
| 통합 패킷 크기 | 65 bytes | 실제 측정값 |
| EFR32BG22 기본 MTU | 250 bytes | AT+MTU 기본값 (매뉴얼 Page 32) |
| iOS 최대 MTU | 185 bytes | 매뉴얼 Page 33 명시 |
| 협상 후 실제 페이로드 | 182 bytes | ATT 헤더 3 bytes 제외 |
| 여유 공간 | 117 bytes | 182 - 65 |
| 1회 전송 가능 여부 | ✅ 가능 | 조건 없이 단일 패킷 전송 |

> ⚠️ **구현 주의사항**: 연결 직후 MTU 협상 완료 콜백 이후에 전송 권장
> (EFR32BG22 매뉴얼 Page 9: "start working after the module is connected for 6s")

## 3. 앱 사전 설정 데이터 정의

고객은 **면세점 방문 전** 앱에 여권 정보와 출국 정보를 등록합니다.
BLE 연결 및 정보 전달은 **계산대 앞에서** 이루어집니다.

### 3.1 여권 정보 (필수)

| 항목 | 설명 | 유효성 검사 | 예시 |
|------|------|------------|------|
| 여권번호 | 영문+숫자 최대 9자 | 필수, 영문+숫자만 허용 | M12345678 |
| First Name | 여권 영문 이름 (대문자) | 필수, 영문만 허용 | JOONHO |
| Last Name | 여권 영문 성 (대문자) | 필수, 영문만 허용 | KIM |
| 여권 만료일 | YYMMDD | 필수, 오늘 이후 날짜 | 300520 |

### 3.2 출국 정보 (선택)

| 항목 | 설명 | 유효성 검사 | 예시 |
|------|------|------------|------|
| 항공사 코드 | IATA 2자리 | 선택, 영문 2자 | KE |
| 편명 | 항공사코드+숫자 | 선택, 영문+숫자 | KE123 |
| 출국시간 | HHMM | 선택, 4자리 숫자 | 1430 |

> ⚠️ **출국 정보 미등록 처리**: Index P4~P6을 빈값으로 전송.
> VPOS에서 감지 후 직원이 탑승권 보고 수동 입력 (Phase 2-B).

## 4. 기능 요구사항 (User Stories)

### 4.1 VPOS 단말기 앱

---

**US-V1: BLE 스캔 및 고객 목록 표시**
```
As a  매장 직원
I want  BLE 스캔으로 Advertise 중인 고객 목록(이름, 전화끝4자리, 신호강도)을 보도록
So that  정확한 고객을 선택하여 연결 가능
```

| 항목 | 내용 |
|------|------|
| 화면 | V-01 메인 대기 → V-02 고객 목록 |
| 트리거 | BLE Scan 시작 (앱 실행 시 자동) |
| 표시 항목 | 이름, 전화번호 끝 4자리, RSSI 신호 강도 바 |
| 정렬 기준 | RSSI 강도 내림차순 (가까운 고객 상단) |
| 갱신 주기 | 1초마다 목록 업데이트 |
| 연결 제한 | 동시 1명만 연결 가능 |

---

**US-V2: 통합 패킷 1회 수신 및 파싱**
```
As a  매장 직원
I want  GATT 연결 직후 '#' 구분 통합 패킷을 수신하고
        멤버십과 여권/출국 정보로 분리 파싱된 결과를 화면에 표시하도록
So that  여권·탑승권 수기 입력 없이 단번에 모든 정보 확인 가능
```

| 항목 | 내용 |
|------|------|
| 화면 | V-03 고객 정보 확인 |
| 트리거 | GATT Write Characteristic 수신 |
| 파싱 로직 | `split("#")` → 멤버십 / 여권·출국 분리 |
| 표시 항목 | 이름, 회원번호, 등급, 포인트, 여권번호, 만료일, 항공편, 출국시간 |
| 만료일 검증 | 오늘 날짜 이후인지 자동 확인 |
| 타임아웃 | 연결 후 10초 이내 수신 없으면 재시도 안내 |

---

**US-V3: 출국 정보 수동 입력 (Fallback)**
```
As a  매장 직원
I want  출국 정보 미등록 시 탑승권 보고 VPOS에 직접 입력하도록
So that  미등록 고객도 동일한 프로세스 진행 가능
```

| 항목 | 내용 |
|------|------|
| 화면 | V-03B 출국정보 수동입력 |
| 트리거 | 통합 패킷 Index P4~P6 빈값 감지 |
| 입력 필드 | 항공사 코드, 편명, 출국시간 |
| 완료 후 | V-04 면세 자격 조회 화면으로 자동 이동 |

---

**US-V4: 면세 자격 조회 (자사 POS 연동)**
```
As a  매장 직원
I want  구두 확인 완료 후 여권번호+항공편을 자사 POS에 등록하여
        면세 자격 및 자사 누적 구매액을 확인하도록
So that  상품 스캔 전 면세 판매 자격 확인 완료 (표준 절차 준수)
```

| 항목 | 내용 |
|------|------|
| 화면 | V-04 면세 자격 조회 결과 |
| 트리거 | [구두 확인 완료] 버튼 터치 |
| 연동 시스템 | 자사 POS 시스템 (내부 API) |
| 전달 데이터 | 여권번호, 항공사코드, 편명, 출국시간 |
| 조회 결과 | 자격 여부 + 자사 당일 누적 구매액 |
| 한도 초과 처리 | $800 초과 시 관세 신고 안내 표시 |
| 오류 처리 | POS 연동 실패 시 재시도 버튼 표시 |

---

**US-V5: 상품 바코드 스캔**
```
As a  매장 직원
I want  면세 자격 확인 완료 후 상품을 바코드 스캔하도록
So that  표준 절차(신원 → 스캔)를 자연스럽게 준수
```

| 항목 | 내용 |
|------|------|
| 화면 | V-05 상품 바코드 스캔 |
| 트리거 | V-04 [상품 스캔으로 이동 →] 터치 |
| 스캔 방식 | 외부 바코드 스캐너 (USB/Bluetooth) |
| 조회 정보 | 상품명, 레퍼런스, 판매가 |
| 진입 조건 | V-04 자격 조회 완료 상태에서만 진입 가능 |

---

**US-V6: 결제 금액 확정 및 오프라인 결제 모듈 인계**
```
As a  매장 직원
I want  혜택 적용 후 최종 결제 금액을 확정하고 카드 단말기로 인계하도록
So that  별도 오프라인 결제 모듈이 이어서 처리 가능
```

| 항목 | 내용 |
|------|------|
| 화면 | V-06 혜택안내·결제금액 확정 |
| 혜택 적용 | VIP 등급별 즉시 할인율 자동 적용 |
| 인계 데이터 | 고객 정보, 상품 정보, 최종 결제 금액, 할인 내역 |
| 인계 방식 | 오프라인 카드 단말기 모듈로 데이터 전달 |

---

**US-V7: BLE 실패 시 수동 전환 (Fallback)**
```
As a  매장 직원
I want  BLE 연결 실패 시 여권·출국 정보를 수동 입력하도록
So that  앱 미사용 고객 또는 BLE 장애 시에도 동일한 프로세스 진행 가능
```

| 항목 | 내용 |
|------|------|
| 화면 | V-08 수동 여권·출국 입력 |
| 트리거 | BLE 연결 10초 타임아웃 또는 [수동 입력] 버튼 |
| 입력 필드 | 여권번호, 이름, 만료일, 항공사, 편명, 출국시간 |

### 4.2 고객 면세점 앱

---

**US-C1: 여권/출국 정보 사전 등록**
```
As a  고객
I want  앱에 여권 정보와 출국 정보를 미리 등록하도록
So that  계산대에서 여권을 꺼낼 필요가 없음
```

| 항목 | 내용 |
|------|------|

### 5.2 BLE 연결 실패

```
감지 조건:  GATT 연결 10초 이내 실패 또는 타임아웃

VPOS 처리:

  → V-08 수동 입력 화면으로 전환
```

### 5.3 BLE 연결 중 끊김 (GATT Disconnected)

```
감지 조건:  onConnectionStateChange(DISCONNECTED) 콜백
고객 앱:    "연결이 끊겼습니다." 토스트 메시지 → C-03 카드 탭으로 복귀
VPOS 처리:
  - 통합 패킷 이미 수신된 경우: 수신 데이터 유지, 이후 프로세스 계속
  - 수신 전 끊김: 목록 화면으로 복귀, 재연결 안내
```

### 5.4 여권 만료 또는 당일 출국편 없음

### 7.2 패킷 파싱 예시 — VPOS Android / Kotlin

```kotlin
val raw = "DF2023091234|V|258000#M12345678|JOONHO|KIM|300520|KE|KE123|1430"

val parts = raw.split("#")
val m = parts[0].split("|")   // 멤버십
val p = parts[1].split("|")   // 여권/출국

val memberNo = m[0]            // DF2023091234

val airline    = p[4]          // KE      (빈값이면 미등록)
val flightNo   = p[5]          // KE123
val depTime    = p[6]          // 1430    (HHMM)

val hasDeparture = airline.isNotEmpty()
```

### 7.3 패킷 파싱 예시 — 고객 앱 Android / Kotlin (전송 측)

```kotlin
// 고객 앱: 연결 완료 후 통합 패킷 조립 및 전송
fun buildUnifiedPacket(): String {
    val memberNo  = prefs.getString("memberNo", "")
    val grade     = prefs.getString("grade", "N")
    val point     = prefs.getInt("point", 0)
    val passportNo = prefs.getString("passportNo", "")
    val firstName  = prefs.getString("firstName", "")
    val lastName   = prefs.getString("lastName", "")
    val expiry     = prefs.getString("expiry", "")   // YYMMDD
    val airline    = prefs.getString("airline", "")  // 미등록 시 ""
    val flightNo   = prefs.getString("flightNo", "")
    val depTime    = prefs.getString("depTime", "")  // HHMM

    return "$memberNo|$grade|$point#$passportNo|$firstName|$lastName|$expiry|$airline|$flightNo|$depTime"
}

// GATT onMtuChanged 콜백 이후 전송
override fun onMtuChanged(gatt: BluetoothGatt, mtu: Int, status: Int) {
    if (status == BluetoothGatt.GATT_SUCCESS) {
        val packet = buildUnifiedPacket()
        writeCharacteristic(packet.toByteArray(Charsets.UTF_8))
    }
}
```
